# 外部文献/GitHub 对比与数据可靠性自检

检查日期：2026-07-04

## 1. 对比目标

本次检查不是默认相信本地处理结果，而是把 `codex版` 当前生成的 HLS/MODIS NDVI、FAI 数据，与太湖蓝藻/漂浮藻遥感文献、公开数据集和 GitHub 开源流程进行对照，判断数据源、处理方法和数值范围是否可信。

## 2. 外部文献和公开数据的共识

### 2.1 太湖蓝藻长期监测常用 MODIS

Hu 等关于太湖蓝藻的经典 MODIS 方法使用 MODIS 645、859、1240 nm 反射率计算 FAI，并客观确定 FAI 阈值约 `-0.004` 来区分 bloom/non-bloom water。该方法明确把 MODIS 作为中等空间分辨率、高时间频率的蓝藻监测数据源。

Zhang 等 2019 在 Google Earth Engine 上做太湖 2000-2018 蓝藻长期监测，也使用 MODIS 和 FAI 流程。该研究指出太湖蓝藻有显著时空差异，西北湖区、湾区更严重，主要从 4 月开始，7-8 月加重，10 月后减弱。

结论：我们使用 MODIS Terra/Aqua 作为每日粗分辨率时间序列层，方向与文献一致。

### 2.2 高分辨率数据通常用于验证或补充，不等于每日连续观测

HLS 官方说明显示，HLS 由 Landsat 8/9 与 Sentinel-2 A/B 构成，提供 30 m NBAR 表面反射率，理论重访为 2-3 天，但实际在太湖梅雨和多云季节受云、云影、传感器覆盖影响很大。

THQBCA 公开数据集也采用多源组合思路：水质、气候、遥感、生物光学、人类活动数据混合组织；卫星派生数据空间分辨率从 30 m 到 500 m 不等，其中 FAC 使用 MODIS 250 m 年度数据，Landsat 30 m 用于水质/植被等年度产品。

结论：我们把 HLS 定位为“30 m 真实观测层”、MODIS 定位为“每日粗分辨率层”是合理的；不能把 MODIS 上采样伪装成每日 30 m 真实观测。

### 2.3 文献更强调 FAC/EBA 或分类面积，而不只输出 NDVI/FAI

较新的太湖数据集和多传感器研究更多使用 FAC（fractional floating algae cover）或 EBA（equivalent bloom area），并用 OLI 等高分辨率观测校正/验证 MODIS、VIIRS、GOCI、OLCI 等粗分辨率数据。

结论：我们当前只输出 NDVI/FAI，还没有完成 FAC/EBA 转换与同日高低分辨率交叉验证，因此只能算指数产品初版，不能直接等同于成熟蓝藻面积数据集。

## 3. GitHub/开源流程对比

GitHub 上可见的通用遥感指数库和 GEE 工具，例如 `awesome-spectral-indices` 与 `Ndvi2Gif`，支持多传感器、多指数、GEE 端合成、GeoTIFF 导出和时间序列分析。它们的共同思路是：

1. 指数公式和传感器波段必须明确；
2. 云/质量掩膜必须按传感器分别处理；
3. 长时间序列常使用 GEE 端批处理或合成；
4. 高分辨率与高时间频率通常需要多源融合，而不是单一卫星直接满足。

结论：我们的整体架构与这些开源实践一致；但还缺少成熟开源流程常见的参数化阈值、传感器间偏差校正、批量可视化 QA 和自动验证图。

## 4. 本地数据与外部结果的数值对比

### 4.1 MODIS 结果

当前本地 MODIS 输出：

- 时间：2024-05-01 到 2024-07-31，共 92 天。
- 数据：MOD09GA Terra + MYD09GA Aqua。
- 分辨率：500 m。
- 自检结果：92 个 GeoTIFF 都存在；CRS 为 EPSG:32651；nodata 已统一为 `-9999`。
- 问题：51 天没有 `QA > 0` 有效像元。

典型日期：

- `2024-05-14`：有效像元 2105 个，按 500 m 像元约 526.25 km² 有效覆盖；若用 MODIS FAI >= 0.0693，则约 119.0 km²。
- `2024-07-30`：有效像元 8914 个，约 2228.5 km² 有效覆盖；若用 MODIS FAI >= 0.0693，则约 265.75 km²。

与文献相比：

- 太湖长期研究中，平均蓝藻水华面积约百平方公里量级，强水华年份/日期可达到更高量级。
- 因此 MODIS 推得的 `119-266 km²` 在数量级上并不离谱。
- 但这些面积目前不是最终蓝藻面积，因为我们没有使用文献同款 Rrc 处理、没有做 FAC/EBA 校正、没有权威湖界裁剪。

### 4.2 HLS 结果

当前本地 HLS 成功下载了 `2024-05-14`：

- 分辨率：30 m。
- CRS：EPSG:32651。
- 有效像元：1,714,542 个，约 1543.09 km² 有效覆盖。
- NDVI >= 0：约 38.64 km²。
- FAI >= 0.00001：约 6.88 km²。
- FAI >= 0.0693：0 km²。

与文献相比：

- HLS 的 NDVI 正值面积几十平方公里，作为高分辨率可疑漂浮藻/植被候选区，数量级合理。
- 但 HLS FAI 数值极小，和 MODIS FAI 量级差别很大。直接套用 Landsat/Sentinel-2 某些文献中 `FAI >= 0.0693` 的阈值会得到 0，这说明当前 HLS FAI 与该阈值不可直接比较。

## 5. 可靠性判断

### 可信部分

1. 数据来源真实：HLS、MODIS 都来自 GEE 官方数据集。
2. 卫星选择方向合理：MODIS 做每日粗分辨率，HLS 做高分辨率观测，与太湖文献和开源流程一致。
3. QA 掩膜有依据：HLS 使用 Fmask，MODIS 使用 `state_1km` 与 `QC_500m`。
4. MODIS 部分日期的候选藻华面积在文献常见数量级内。

### 高风险部分

1. HLS 与 MODIS 的 FAI 量级明显不一致，不能直接共用阈值。
2. 当前 MODIS FAI 使用 SWIR1 约 1640 nm，而 Hu 等 MODIS 太湖经典方法使用 1240 nm；这可能是主要偏差来源之一。
3. 当前使用表面反射率 SR 计算 FAI，而部分文献使用 Rayleigh-corrected reflectance Rrc，指数阈值不能无条件迁移。
4. 51 天 MODIS 无有效像元，说明“92 天输出文件”不等于“92 天有效观测”。
5. 当前 AOI 是 bbox + JRC 稳定水体，不是权威太湖湖界，会影响面积统计。

## 6. 下一步改正建议

1. MODIS 增加 `FAI_1240`：使用 `sur_refl_b05` 作为 SWIR 基线波段，保留当前 `FAI_1640`，二者并行输出。
2. HLS 不直接套 `0.0693` 阈值：先用直方图双峰法或 Otsu/高斯混合模型做日期自适应阈值。
3. 做同日交叉验证：以 `2024-05-14`、后续 `2024-05-17`、`2024-07-31` 为样本，把 HLS 聚合到 500 m，与 MODIS 像元比较 NDVI、FAI 和候选藻华面积。
4. 引入 FAC/EBA：参考 THQBCA 与 Xue 等多传感器 FAC 方法，不只输出二值藻华面积。
5. 替换 AOI：使用权威太湖湖界和分湖区矢量，重新计算覆盖率和面积。
6. 输出两个产品层：`observed` 表示真实有效观测，`gap_filled` 表示插值/融合估算，二者必须分开。

## 7. 结论

当前数据源和总体流程方向是可信的，但当前产品还不能作为最终蓝藻面积结果。最大的问题不是“数据是假”，而是“跨传感器 FAI 没有校正、MODIS 波段选择可能不贴近经典太湖方法、HLS 有效天数少、AOI 不够正式”。因此目前结果适合作为初版指数产品和流程验证，不适合直接写成最终科学结论。
