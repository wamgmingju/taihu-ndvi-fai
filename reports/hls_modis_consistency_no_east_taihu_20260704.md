# HLS-MODIS 同日一致性检查

日期：2026-07-04

## 1. 检查目的

当前每日主产品是 MODIS v2 去东太湖日序列。HLS 只有少数高分辨率有效日期，因此不能作为每日主序列，但可以用来检查 MODIS 在同日同区域是否存在系统偏差。

本检查把 HLS 30 m 有效像元聚合到 MODIS 500 m 像元范围内，再与同日 MODIS v2 真观测像元比较。

输出表：

```text
outputs/tables/hls_modis_consistency_no_east_taihu.csv
```

脚本：

```text
scripts/19_compare_hls_modis_consistency.py
```

## 2. 数据和配对方法

HLS 数据：

```text
outputs/rasters/hls_observed_no_east_taihu/
```

MODIS 数据：

```text
outputs/rasters/modis_daily_v2_fai1240_no_east_taihu/
```

方法：

1. 读取 HLS `NDVI`、`FAI`、`QA`。
2. 读取 MODIS v2 `NDVI`、`FAI_1240`、`QA`。
3. 只保留 QA 有效像元。
4. 对每个 MODIS 500 m 有效像元，统计其覆盖范围内的 HLS 30 m 有效像元均值。
5. 计算 MODIS 与 HLS 的均值、偏差、MAE、RMSE、相关系数。

注意：HLS 的 FAI 与 MODIS `FAI_1240` 使用的波段基线不同，FAI 对比只能作为一致性参考，不能直接按绝对值等同。

## 4. 判断

NDVI 的检查说明：

- 2024-05-14 的配对像元最多，924 个，NDVI 相关性 0.7883，RMSE 0.0750。这一天可以作为当前最可靠的 HLS-MODIS 校正参考。
- 2024-05-17 配对像元只有 101 个，NDVI RMSE 达 0.3531，相关性 0.4860，说明这一天空间代表性差，不能单独用于强校正。
- 2024-07-31 配对像元 138 个，相关性 0.8033，但 HLS 覆盖率低，只能作为辅助参考。
- 3 个日期都显示 MODIS NDVI 比 HLS 聚合值偏高，偏差约 0.065-0.077。这可能来自传感器波段、观测时刻、BRDF/大气残差、混合像元和 QA 差异。

FAI 的检查说明：

- FAI 相关性在 0.6673-0.8556 之间，说明空间变化方向有一定一致性。
- 但 HLS FAI 和 MODIS `FAI_1240` 的绝对量级差异明显，不能直接用 HLS FAI 均值去判定 MODIS FAI_1240 绝对值错误。
- 如果后续要做 FAI 校正，应使用同日样本建立经验映射，而不是直接相减。

## 5. 对当前每日产品的影响

当前 MODIS v2 每日 NDVI/FAI 产品仍可作为 92 天每日主序列，但必须保留以下限制：

- NDVI 可能存在约 0.07 的 MODIS-HLS系统偏高。
- 2024-05-14 是最可信的校正日；2024-05-17 和 2024-07-31 只能辅助判断。
- FAI 不能直接跨传感器用绝对值硬校正，需要先做经验关系或阈值重标定。

## 6. 建议修正

下一步建议：

1. 暂不改变已经生成的 92 天原始 MODIS v2 和 gap-filled 文件，避免破坏数据溯源。
2. 新增一个校正版产品目录，而不是覆盖原产品。
3. NDVI 可先试验一个保守校正：`NDVI_corrected = NDVI - 0.065`，并在报告中标记这是基于 2024-05-14 的一阶偏差修正。
4. FAI 暂不做简单常数校正，应先用配对像元建立线性关系并检查阈值稳定性。
5. 每日 PNG 预览应同时显示 `STATUS`，否则用户会误把插值和缺失当作真实观测。
