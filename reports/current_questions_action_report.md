## 2. HLS 有效覆盖率 > 0.15 是什么意思，为什么理论 2-3 天重访却只有 3 天可用

`有效覆盖率 > 0.15` 的意思是：在太湖水体范围内，经过云、云影、邻近云、雪冰、高气溶胶、异常反射率等掩膜后，还剩下的有效 HLS 像元面积占水体面积的比例大于 15%。

HLS “2-3 天有一景”指的是卫星重访/有影像，不等于“太湖水面有足够清晰可用像元”。这中间会被几类东西卡掉：

1. 云和云影。5-7 月太湖多云、梅雨，光学卫星很容易被云挡住。
2. HLS Fmask。Fmask 会去掉云、邻近云、云影、雪冰、气溶胶高值像元。
3. 水体掩膜。只有落在太湖水体范围内的像元才算有效。
4. 反射率有效性检查。红、绿、近红外、SWIR 必须在合理范围内。
5. 单日合成限制。同一天可能有影像，但覆盖太湖的清晰区域很小。

能不能放松？可以，但我不建议无原则放松。我的独立判断是：

- HLS 的 `0.15` 阈值适合作为“下载整幅高分图”的门槛。
- 可以增加一个 `0.05-0.15` 的“低覆盖观测层”，用于局部验证，不用于全湖面积统计。
- 不建议把低于 `0.05` 的 HLS 当作全湖图，因为那基本只是局部碎片。

## 3. MODIS 51 天缺的 QA 是什么，QA 有什么用

QA 是 quality assurance，质量控制标记。我们虽然在做 NDVI/FAI，但 NDVI/FAI 只有在像元真实、清晰、反射率可信时才有意义。云、云影、厚气溶胶、传感器质量差的像元也能算出 NDVI/FAI，但那些数值可能是假的。

在当前 MODIS 流程里，`QA > 0` 表示该像元通过了云、云影、气溶胶、卷云、内部云、邻近云、QC 等检查，并被保留下来。51 天没有 `QA > 0` 有效像元，意思是这些天在当前严格规则下没有可信 MODIS 水体观测。

我的判断：QA 不是最终研究对象，但它是判断 NDVI/FAI 是否可信的门。没有 QA，NDVI/FAI 很容易被云当作藻华。

## 4. Sentinel-3 OLCI 有什么用

Sentinel-3 OLCI 是水色传感器，空间分辨率约 300 m，适合看叶绿素、蓝藻、水色变化，时间频率也比 Landsat/Sentinel-2 高。它理论上很适合补 MODIS 和 HLS 之间的空档。

但问题是：GEE 中常用的 Sentinel-3 OLCI 数据多是 L1 TOA radiance，不是已经处理好的水体反射率或蓝藻产品。直接拿 L1 辐亮度算 NDVI/FAI，物理意义不稳。

我的判断：OLCI 有用，但不要急着直接混进当前 NDVI/FAI。若要用，应找 L2 水色产品、CI/FAC 产品，或单独做水色大气校正。

## 5. 卫星处理方法不同，为什么还要“统一”

统一不是说所有卫星用同一个云掩膜或同一个公式参数。不同卫星当然要按各自 QA、波段、尺度处理。

这里说的统一是统一输出语义：

- 每个像元都要知道来源卫星；
- 每个像元都要知道是真观测还是插值；
- 每个像元都要知道可信度；
- 每个指数都要说明使用了哪个波段和反射率产品。

也就是说，处理方法可以不同，输出标准必须统一。否则后面没法合并、对比、验证。

## 11. 人工样本校准是什么，有什么用

人工样本校准不是随便画，而是在高分影像上选一些明显类别：

- 明显蓝藻/漂浮藻；
- 清洁水体；
- 云/薄云；
- 岸边混合像元；
- 水草或其他水生植被。

然后统计这些样本的 NDVI/FAI 分布，找出能把蓝藻和非蓝藻分开的阈值或模型。

作用：

1. 避免直接套别人年份、别人传感器、别人反射率产品的阈值。
2. 解释为什么某个阈值合理。
3. 为 HLS 和 MODIS 之间的校正提供锚点。

但你现在决定先只做 NDVI 和 FAI，我同意。人工样本可以放到后面验证阶段。

## 15. 类似文献怎么做

两篇本地 PDF 给出的参考很直接：

1. 2019 MODIS + GEE 文献：
   - 使用 MODIS 日数据；
   - 红/NIR 来自 MOD09GQ 250 m，SWIR 来自 MOD09GA 重采样；
   - 使用湖界/陆地掩膜，并向水体方向腐蚀一像元；
   - 使用云掩膜；
   - 使用 Rayleigh-corrected reflectance；
   - FAI 阈值采用 `> -0.004`。

2. 2022 Landsat + GEE 文献：
   - 使用 Landsat 5/7/8 Level-2 surface reflectance；
   - 云量筛选 `<20%`；
   - GEE 完成云去除、裁剪、合成；
   - 使用 FAI 阈值提取蓝藻；
   - 用 Sentinel-2 同期高分影像和其他研究结果做验证。

对我们的启发：MODIS 主序列 + 高分影像验证是合理路线；湖界、岸边腐蚀、云掩膜、反射率产品和阈值来源必须写清楚。

## 16. 缺失怎么处理，时间插值和多源融合是什么意思

你理解得对：缺失处理就是有些日期/像元没有真实观测，要不要补。

本轮已先做时间插值：

- 输入：MODIS 真实观测层。
- 方法：同一个像元，前后都有真实观测时，按时间线性插值。
- 限制：默认最近真实观测超过 10 天就不补。
- 输出：`outputs/rasters/modis_daily_gapfilled_linear/`。

结果：

- 真实观测像元：64,655。
- 插值补充像元：462,793。
- 仍缺失像元：2,570,468。

多源融合不是简单把所有卫星重采样到 30 m 后合成一张“看起来高分”的图。正确理解是分层：

- HLS/S2/Landsat：真实高分观测层，空间细，但时间不连续。
- MODIS/Sentinel-3：粗分辨率日序列层，时间密，但空间粗。
- 同日/近同日校正：用高分层校准粗分辨率层的偏差。
- gap-filled 层：在清楚标记不确定性的前提下，用时间插值或多源信息补缺失。

所以不是把 MODIS 硬变成真实 30 m，而是建立“高分观测 + 日序列 + 校正/插值”的分层产品。

## 17. 不确定哪里可靠哪里不可靠，怎么表达

最简单的方式就是增加状态/可信度标记。插值层里我已经用了：

- `STATUS = 30`：真实 MODIS 观测；
- `STATUS = 55`：线性时间插值；
- `STATUS = 90`：仍缺失。

还有一个 `NEAREST_OBS_GAP_DAYS` 波段，表示插值像元离最近真实观测差几天。差 1-2 天比差 10 天可靠。
